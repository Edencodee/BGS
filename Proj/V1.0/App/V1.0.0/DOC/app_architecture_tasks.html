0. 概览

核心检测链路

  1. PLS_SO 进入 EXTI5_9（双边沿），仅置位采样标志
  2. 主循环触发 MeasureTask，按 PLS_SO 电平选择 A/B DMA 缓冲区
  3. Timer2 触发 ADC，DMA 采样 SAMPLE_POINT 点（默认 10）
  4. DMA 完成中断置位 g_isSampleDone，进入 AnalyzeTask
  5. AnalyzeTask：FastTrimmedMean 后计算 V_A、V_B 与差分 V_OUT
  6. OutputTask：与 V_SET 比较得到 presence，驱动 NPN 输出
  7. LedTask：Light/Dark 模式显示工作灯

关键参数与信号

  • V_SET：阈值（mV），Flash 保存
  • led_mode：Light-ON / Dark-ON（Flash 保存）
  • PLS_SO：A/B 相位标志，约 10kHz 翻转
  • SH_OUT：模拟输出，ADC 采样
原则：presence 是物理检测语义；Light/Dark 只影响输出映射与指示灯显示。

2. 采样时序与数据流

Timer2 设定为 100kHz 触发 ADC（48MHz / 480 = 100kHz），SAMPLE_POINT=10，单次采样约 100us。 PLS_SO A/B 相位约 100us，对应一组采样。

流程图（时序）

当前实现要点

  • MeasureTask 读取 PLS_SO 电平选择 A/B buffer
  • ADC_StartSampling 启动 DMA 采样，完成后由 DMA ISR 置位 g_isSampleDone
  • AnalyzeTask 做去极值平均，计算 V_A、V_B 与 V_OUT
MeasureTask:
  if PLS_SO == High -> use B buffer
  else              -> use A buffer
  ADC_StartSampling()

3. 主循环与中断协作

主循环（superloop）

while (1) {
  if (g_isSysTickInt) { SysTickTask(); }
  if (g_isStartSampling) { MeasureTask(); }
  if (g_isSampleDone) {
    AnalyzeTask();
    OutputTask();
    LedTask();
  }
}
EXTI 只置位 g_isStartSampling，DMA ISR 只置位 g_isSampleDone；计算工作全部在主循环完成。

中断职责

中断动作
SysTick置位 g_isSysTickInt，提供 10ms 按键节拍
EXTI5_9 (PLS_SO)若上次采样完成，则置位 g_isStartSampling
DMA CH0采样完成，置位 g_isSampleDone

7. 需要注意的实现点

  • 需求为 V = V_B - V_A 且 V <= V_SET 判“检测到目标”，当前实现为 V_A - V_B 且 V_OUT >= V_SET。
  • Light/Dark 当前只影响工作灯，不影响 NPN 输出映射。
  • main.c 中使用 exti_software_interrupt_enable 进行测试，量产需确认是否移除。
  • g_param_cfg 为 volatile，全局读写需注意中断/主循环并发访问。
  • 输出响应 5ms，当前采样/分析链路可满足，但需确认滤波窗口与主循环负载。

中断服务流程

  • 事件传递：主循环轮询标志，调用任务处理

硬件初始化序列（按时间顺序）

  2. GPIO 初始化（LED/NPN/BTN/PLS_SO/ISL700 IO）
  3. PLS_SO EXTI 配置（双边沿）
  4. ADC + DMA + Timer2 配置
  5. ParamCfg_Init 读取 NVM 默认参数
  6. ParamCfg_ApplyToIC 配置 ISL700_2A

10. 应用层详细设计

API 接口规范（当前工程）

函数参数返回值调用方式
MeasureTask()无void异步（由 EXTI 标志驱动）
AnalyzeTask()无void同步（主循环）
OutputTask()无void同步（主循环）
ParamCfg_Save()parambool同步（调用方等待）

当前工程未使用回调机制，采用标志位 + 主循环调度。

业务逻辑层状态机

  • 空闲 → 检测中：PLS_SO 边沿触发 g_isStartSampling
  • 检测中 → 触发：DMA 完成后 AnalyzeTask 计算完成
  • 触发 → 空闲：完成 OutputTask / LedTask 后回到等待
  • 错误恢复：当前为占位，未实现统一错误恢复策略

阈值管理逻辑

  • 阈值 V_SET 存放在 g_param_cfg.app.v_set
  • OutputTask 直接比较 V_OUT 与 V_SET
  • 动态修改通过 ParamCfg_SetVset / ParamCfg_Save

多传感器协同

当前工程为单通道单传感器设计，无多传感器协同机制。

配置管理方案

  • 默认参数：PARAM_CONFIG_DEFAULT
  • 运行时修改：ParamCfg_SetApp / ParamCfg_SetIC / ParamCfg_Save
  • 掉电保存：NVM A/B 页 + CRC32
  • 合法性检查：目前为隐式（未做范围校验）

校准算法

  • 当前无自动基线校准
