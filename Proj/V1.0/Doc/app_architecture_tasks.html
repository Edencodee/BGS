0. 概览

核心检测链路

  1. ADC 由 TIM 触发采样 SH_OUT（100kHz 起步）
  2. DMA circular 写入 raw_ring[RAW_LEN]，产生 half/full 事件
  3. PLS_SO 双边沿 EXTI：每次边沿记录 pos = (RAW_LEN - CNT)%RAW_LEN + 相位
  4. Task_Meas 按 toggle 事件切分 block，把样本分拣到 A_buffer/B_buffer
  5. 每 5ms 统计得到 VA/VB，计算 V=VB-VA，得到 presence_raw
  6. Task_Output 稳定化 presence，按 Light/Dark 映射 output_on，驱动 NPN（ON=GPIO=0）

输出与人机

  • output_mode（Light-ON / Dark-ON）由按键切换，且需要 Flash 掉电保存
  • 工作灯建议显示 output_on（语义与 Light/Dark 一致）
  • 电源灯常亮
原则：presence 是物理检测语义（不变）；Light/Dark 只影响输出映射，不改变检测本身。

2. 采样时序与切换（重点）

目标：使用 PLS_SO 的真实边沿作为 A/B 相位边界，不依赖固定 100us。 EXTI 双边沿 ISR 里读取 DMA 的 CNT，换算出边沿发生时 raw 环形缓冲的写入位置 pos， 将 pos 作为切分点，供 Task_Meas 在块处理中做精确分段与分拣。

2.1 时序示意图（ASCII）

时间轴 --->

PLS_SO:  AAAAAAA|BBBBBBB|AAAAAAA|BBBBBBB|....
               ^       ^       ^       ^
               |       |       |       |
             EXTI    EXTI    EXTI    EXTI      （双边沿：每次翻转都触发）

ADC采样:  . . . . . . . . . . . . . . . .     （TIM触发，周期Ts=10us @100kHz）

DMA raw:  [s0 s1 s2 ... s99] [s100 ... s199]   （circular）
           ^ half event        ^ full event

2.2 边沿位置 pos 的计算（在 EXTI ISR 中完成）

EXTI ISR 最小动作 必须很短

// 1) 读 DMA 剩余传输数
cnt = dma_transfer_number_get(adc_dma_ch);

// 2) 折算当前 raw 写入位置（近似）
pos = (RAW_LEN - cnt) % RAW_LEN;

// 3) 读 PLS_SO 当前电平 -> 得到新相位（A/B）
level = gpio_input_bit_get(PLS_SO_PORT, PLS_SO_PIN);
phase = (level ? PHASE_B : PHASE_A);   // 具体映射按你实测

// 4) 入队：toggle_event = {pos, phase, time_ms}
toggle_queue.push(toggle_event);
注意：pos 可能与“恰好对应的那个样本”差 0~1 点，这没关系：在 Task_Meas 分段时丢弃每段首尾 N_discard 点即可吸收误差。

2.3 Task_Meas 如何用 toggle 切分一个 DMA block

输入：block = raw[start..start+LEN-1]，以及此范围内的 toggle_pos 列表
输出：若干 segment，每个 segment 属于 A 或 B，相邻 segment 在 toggle_pos 处分界

例：block 覆盖 pos 0..99，其中 toggle 发生在 pos=23, 51, 78
segments:
  [0..22] phase=A
  [23..50] phase=B
  [51..77] phase=A
  [78..99] phase=B
好处：当 A/B 周期变长或变短时，toggle_pos 间距会改变，segments 会自动变长/变短，软件无需改“每相位多少点”的假设。

2.4 分拣到 A_buffer/B_buffer（并丢弃首尾点）

对每个 segment:
  seg_len = end - begin + 1
  if seg_len <= 2*N_discard + N_min_valid: 丢弃（太短）
  else:
    copy segment[begin+N_discard .. end-N_discard] -> A_buffer 或 B_buffer

建议：100kHz 初版 N_discard=1（段首尾各丢 1 点）。后续采样率提高时，用“按时间折算点数”的方式调整。

3. 事件与调度（superloop）

推荐事件位

EVT_ADC_HALF      // DMA half complete
EVT_ADC_FULL      // DMA full complete
EVT_SYSTICK_1MS   // 1ms tick
EVT_KEY_TICK      // 10ms tick
EVT_LED_TICK      // 10ms tick
EVT_PARAM_TICK    // 20~50ms tick
EVT_FRAME_READY   // 5ms frame produced by Task_Meas
EVT_OUTPUT_UPDATE // optional: immediate refresh after mode switch

主循环执行顺序（建议）

while(1){
  evt = App_FetchEvents();

  Task_Init();           // until RUN

  if(evt & EVT_KEY_TICK)      Task_Key();
  if(evt & (EVT_ADC_HALF|EVT_ADC_FULL)) Task_Meas();
  if(evt & (EVT_FRAME_READY|EVT_SYSTICK_1MS|EVT_OUTPUT_UPDATE)) Task_Output();
  if(evt & (EVT_LED_TICK|EVT_OUTPUT_UPDATE)) Task_LED();
  if(evt & EVT_PARAM_TICK)    Task_Param();

  App_Idle();            // WFI / watchdog etc.
}
建议：ISR 只置位事件/记录 toggle；重计算全部放在 Task 中完成。

4. 各 Task 需要实现的函数与步骤

Task_Init 上电状态机

目标：参数加载 → IC 配置 → 启动采样 → RUN

需要实现的接口：

void Task_Init(void);
bool App_IsRunning(void);

bool ParamSrv_Load(AppParam* p);
bool IcCfgSrv_Apply(void);

void MeasSrv_Start(void);
void MeasSrv_Stop(void);

步骤：

  1. 读取 Flash 参数（V_SET、output_mode）
  2. BitBang 配置 IC（可重试）
  3. 启动 TIM/ADC/DMA + 使能 PLS_SO EXTI

Task_Key Light/Dark 切换

触发：10ms tick（软件去抖）

需要实现的接口：

void Task_Key(void);
void KeySrv_Scan10ms(void);
bool KeySrv_GetShortPress(void);

void ParamSrv_RequestSave(void);

步骤：

  1. 扫描按键，去抖
  2. 短按：切换 output_mode（LIGHT_ON <-> DARK_ON）
  3. 请求保存（延时 2s 再写 Flash）

Task_Meas 采样与分拣核心

触发：DMA half/full

需要实现的接口：

void Task_Meas(void);

bool MeasSrv_GetBlock(MeasBlock* blk);   // blk: ptr,len,range
bool MeasSrv_PopToggle(ToggleEvent* ev); // pos,phase,time

void MeasSplitter_ProcessBlock(const MeasBlock* blk);
bool MeasFrame_IsReady(void);
void MeasFrame_Get(MeasFrame* frame);

步骤（每 block）：

  1. 取 raw block（1ms / 100 点）
  2. 收集 block 范围内的 toggle_pos 列表（可能多个）
  3. 按 toggle_pos 切分为 segments（每段属于 A 或 B）
  4. 每段丢弃首尾 N_discard 点，把剩余点写入 A_buffer/B_buffer
  5. 累计到 5ms 窗口：计算 VA/VB、V、presence_raw，置 EVT_FRAME_READY
关键点：segment 切分依赖 toggle_pos（来自 CNT），而不是“每 10 点切一次”。这样周期变化时自动适配。

Task_Output 稳定化 + 输出驱动

触发：frame_ready（推荐）或 1ms tick

需要实现的接口：

void Task_Output(void);

void OutputDrv_SetOn(bool on); // on=1 => GPIO=0 (NPN导通)
bool OutputDebounce_Update(bool presence_raw);

步骤：

  1. presence_raw 防抖/迟滞（建议 2~3ms）
  2. 得到 presence（物理语义）
  3. 按 output_mode 映射：Light-ON: output_on=presence；Dark-ON: output_on=!presence
  4. 驱动 NPN：output_on=1 → GPIO=0

Task_LED 指示灯

触发：10ms tick

需要实现的接口：

void Task_LED(void);
void LedDrv_SetPwr(bool on);
void LedDrv_SetWork(bool on);

建议规则：

  • 电源灯：常亮
  • 工作灯：显示 output_on（与 Light/Dark 一致）

Task_Param 掉电保存

触发：20~50ms tick

需要实现的接口：

void Task_Param(void);

void ParamSrv_RequestSave(void); // set dirty + deadline
bool ParamSrv_SaveNow(const AppParam* p);

推荐策略：

  1. dirty 后延时 2s 才写（避免频繁擦写）
  2. 写 Flash 前：短暂停止测量（TIM/ADC/DMA + EXTI）
  3. 写完恢复测量
原因：Flash 写可能关中断且耗时未知，停采样可避免 DMA/EXTI 丢事件导致测量状态错乱。

5. 参数保存与停采样策略

保存内容

  • output_mode（必须保存）
  • V_SET（若你也要掉电保存，可一并保存）

建议 Param 结构

struct AppParam{
  uint32_t magic;
  uint16_t version;
  int32_t  V_SET;
  uint8_t  output_mode; // 0=LIGHT_ON, 1=DARK_ON
  uint32_t crc;
};
