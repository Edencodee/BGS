NVM 新增参数操作指南（精简版）

适用于当前工程的 nvm_param_simple 方案（A/B 双页、整结构体存储、CRC32 校验）。

一、修改结构体

1. 在 sensor_param_t 中新增字段

位置：BSP/inc/nvm_param_simple.h
示例：新增 uint8_t mode_ext;

2. 重新计算结构体大小

  • sensor_param_t 为 packed：字段大小直接相加
  • 记录长度公式：record_size = 4(magic) + 4(seq) + sizeof(sensor_param_t) + 4(crc32) + padding
  • 目标：record_size 必须是 8 字节倍数

3. 调整 padding 大小

计算：padding = (8 - (record_size % 8)) % 8
并同步更新注释。

二、更新默认值

位置：BSP/src/nvm_param_simple.c
为新增字段设置合理默认值。

三、CRC 覆盖范围（关键）

  • CRC 只覆盖：magic + seq + param
  • 不包含 crc32 字段，不包含 padding
建议使用固定长度常量： 
#define NVM_CRC_DATA_LEN  (sizeof(uint32_t) + sizeof(uint32_t) + sizeof(sensor_param_t))

四、兼容旧数据（可选但建议）

方式一：增加版本号字段

  • 新增 uint8_t version;
  • 读取旧数据时按 version 进行兼容填充

方式二：升级 MAGIC

  • 修改 NVM_MAGIC（例如 0x5A5AA55B）
  • 强制重新初始化默认值

五、测试步骤

  1. 确认 sizeof(sensor_param_t) 变化
  2. 确认 sizeof(nvm_record_t) 为 8 字节倍数
  3. 运行最小读写测试
  4. 断电重启验证持久化

六、快速检查清单

  • 新增字段已加入 sensor_param_t
  • padding 已重新计算
  • FACTORY_DEFAULT 已补齐
  • NVM_CRC_DATA_LEN 不包含 padding/crc32
  • 最小用例通过

示例（新增 1 字节字段）

// sensor_param_t 增加字段
uint8_t  mode_ext;

// 重新计算 record_size
// 4 + 4 + (原param_size + 1) + 4 + padding

// 调整 padding 为新的值
