🏗️ 数据结构设计

1. 传感器参数结构体（用户可自定义）

typedef struct { uint8_t pls; /* [4:11] 发射脉冲幅值 */ uint16_t fine_gain; /* [14:23] 细调增益 */ uint8_t coarse_gain1; /* [29:32] 粗调增益1 */ uint8_t work_mode; /* 工作模式 */ uint8_t sample_rate; /* 采样率(kHz) */ uint16_t threshold_high; /* 高阈值 */ uint16_t threshold_low; /* 低阈值 */ } __attribute__((packed)) sensor_param_t; /* 11 字节 */

2. NVM 存储记录（自动补齐到 8 字节倍数）

typedef struct { uint32_t magic; /* 0x5A5AA55A 魔术字 */ uint32_t seq; /* 序号，越大越新 */ sensor_param_t param; /* 参数数据 (11 字节) */ uint32_t crc32; /* CRC 校验 */ uint8_t padding[5]; /* 补齐到 24 字节（8的倍数） */ } __attribute__((aligned(8))) nvm_record_t; /* 24 字节 */
⚠️ 重要： 修改 sensor_param_t 后，必须调整 padding 大小，保证总大小是 8 的倍数！

计算公式：
总大小 = 4(magic) + 4(seq) + sizeof(param) + 4(crc32) + padding
确保：(总大小 % 8) == 0

3. Flash 分区布局

页号 地址 大小 用途
0 ~ 61 0x08000000 ~ 0x0800F7FF 62 KB 应用程序代码
62 (A 页) 0x0800F800 1 KB NVM 参数存储 A
63 (B 页) 0x0800FC00 1 KB NVM 参数存储 B

🛡️ 掉电保护原理

A/B 双页写入流程

初始状态： A 页：seq=10, 有效 B 页：seq=9, 有效 写入新数据： 1. 读取 A/B 页，比较 seq 2. 识别 B 页是旧页（seq=9） 3. 擦除 B 页 4. 更新 seq=11，计算 CRC32 5. 写入 B 页 结果状态： A 页：seq=10, 有效（备份） B 页：seq=11, 有效（最新）

掉电场景分析

掉电时刻 A 页状态 B 页状态 恢复策略
擦除前 ✅ 有效 (seq=10) ✅ 有效 (seq=9) 加载 A 页（seq 更大）
擦除中 ✅ 有效 (seq=10) ❌ 已擦除 加载 A 页（唯一有效）
写入中 ✅ 有效 (seq=10) ⚠️ 部分写入 B 页 CRC 失败，加载 A 页
写入后 ✅ 有效 (seq=10) ✅ 有效 (seq=11) 加载 B 页（seq 更大）
🎯 核心保证： 任何时刻掉电，至少有一页完整有效的数据可用！

判断逻辑：
1. 检查 magic 是否为 0x5A5AA55A
2. 验证 CRC32 是否正确
3. 选择 seq 较大且有效的页
4. 若两页都有效，选 seq 大的；若只有一页有效，选该页

📏 如何新增/修改参数

3 步完成参数扩展

步骤 1：修改 sensor_param_t 结构体

/* 在 nvm_param_simple.h 中 */ typedef struct { uint8_t pls; uint16_t fine_gain; uint8_t coarse_gain1; uint8_t work_mode; uint8_t sample_rate; uint16_t threshold_high; uint16_t threshold_low; /* 新增字段 */ uint8_t new_field1; /* 新参数 1 */ uint16_t new_field2; /* 新参数 2 */ } sensor_param_t; /* 现在是 14 字节 */

步骤 2：调整 nvm_record_t 的 padding

/* 计算总大小 */ 总大小 = 4(magic) + 4(seq) + 14(param) + 4(crc32) = 26 字节 /* 补齐到 8 的倍数：32 字节 */ 需要 padding = 32 - 26 = 6 字节 /* 修改结构体 */ typedef struct { uint32_t magic; uint32_t seq; sensor_param_t param; /* 14 字节 */ uint32_t crc32; uint8_t padding[6]; /* 改为 6 字节 */ } nvm_record_t; /* 32 字节，8 的倍数 ✓ */

步骤 3：更新出厂默认值

/* 在 nvm_param_simple.c 中 */ const sensor_param_t FACTORY_DEFAULT = { .pls = 8, .fine_gain = 512, .coarse_gain1 = 8, .work_mode = 0, .sample_rate = 100, .threshold_high = 50000, .threshold_low = 10000, /* 新字段的默认值 */ .new_field1 = 0, .new_field2 = 100, };
⚠️ 注意事项：
1. 修改参数后，旧的 Flash 数据会因为大小不匹配而失效
2. 首次烧录新固件时，会自动使用出厂默认值
3. 如需保留旧数据，需手动实现版本迁移逻辑

📐 8 字节对齐处理

为什么需要 8 字节对齐？

GD32C231 Flash 写入约束：

  • 必须按 8 字节（doubleword）为单位写入
  • 写入地址必须 8 字节对齐
  • 不足 8 字节的部分自动补 0xFF

自动对齐实现

/* flash_write_buffer() 自动处理对齐 */ /* 1. 检查地址对齐 */ if (addr % 8 != 0) { return -1; /* 地址必须 8 字节对齐 */ } /* 2. 每次处理 8 字节 */ while (offset < len) { uint8_t temp[8] = {0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF}; uint32_t copy_len = (len - offset >= 8) ? 8 : (len - offset); /* 3. 拷贝实际数据，剩余部分保持 0xFF */ memcpy(temp, data + offset, copy_len); /* 4. 手动组装 64 位数据（避免未对齐访问） */ uint64_t doubleword = ((uint64_t)temp[0] << 0) | ((uint64_t)temp[1] << 8) | ((uint64_t)temp[2] << 16) | ((uint64_t)temp[3] << 24) | ((uint64_t)temp[4] << 32) | ((uint64_t)temp[5] << 40) | ((uint64_t)temp[6] << 48) | ((uint64_t)temp[7] << 56); /* 5. 写入 Flash */ fmc_doubleword_program(addr + offset, doubleword); offset += 8; }

示例：写入 24 字节记录

字节范围 内容 操作
0~7 magic + seq (前 4 字节) 第 1 次写入（8 字节）
8~15 seq (后 4 字节) + param (前 4 字节) 第 2 次写入（8 字节）
16~23 param (剩余 7 字节) + crc32 (1 字节) 第 3 次写入（8 字节）

❓ 常见问题 FAQ

Q1: 为什么推荐 A/B 双页而不是单页？

A: A/B 双页只增加 50 行代码和 1KB Flash，但可靠性提升巨大。单页方案在擦除/写入过程中掉电会导致数据完全丢失，而双页方案任何时刻都有至少一份完整数据。

Q2: Flash 擦写次数有限制吗？

A: GD32C231 Flash 典型擦写次数为 10 万次。建议：

  • 不要过于频繁保存参数（如每秒保存）
  • 参数变化时才保存，避免无谓写入
  • 正常使用场景下，10 万次足够使用数十年

Q3: 如何判断 Flash 数据是否有效？

A: 三重检查机制：

  1. Magic 检查：验证 magic 是否为 0x5A5AA55A
  2. CRC32 校验：验证数据完整性
  3. Seq 比较：选择序号较大的有效页

Q4: 修改参数后如何保留旧数据？

A: 精简版不支持自动版本迁移。如需保留旧数据：

  1. 在初始化时读取旧数据
  2. 手动提取需要的字段
  3. 填充到新结构体并保存

Q5: 如何调试 Flash 读写问题？

A: 调试技巧：

  • 使用 J-Link 读取 Flash 地址 0x0800F800 和 0x0800FC00
  • 检查 magic 是否为 0x5A5AA55A
  • 检查 seq 的值是否递增
  • 计算 CRC32 并与保存的值对比

Q6: 能否支持更多参数？

A: 可以，只要总大小不超过 1KB（页面大小）。建议：

  • 单条记录控制在 256 字节以内
  • 使用 packed 属性减少结构体大小
  • 确保总大小是 8 的倍数
