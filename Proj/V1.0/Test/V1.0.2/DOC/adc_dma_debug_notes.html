2问题现象与根因分析

P0

程序只运行一次就"卡死"

现象：上电后完成一次 AnalyzeTask，之后主循环永远不再进入分析。

🔴 根因
EXTI 配置为 EXTI_TRIG_RISING（只有上升沿）。主循环极快， MeasureTask 执行时 PLS_SO 仍为 HIGH，每次都只填 g_B_Buffer， g_isA_Done 永远不置位，g_isA_Done && g_isB_Done 永远不成立。
⚠️ 当时的错误"解法"
加 delay_1ms(5)，让主循环晚执行几毫秒，PLS 电平碰巧已跳低，属于 时序依赖的巧合，不可靠。
P1

两路数据整组错乱（A路数据出现在B Buffer中）

触发条件：注释 delay 后几乎消失；加 delay_1ms(20) 后频繁复现。

根因 A：EXTI 只监听上升沿，A路相位（下降沿）根本不触发中断

// 错误
exti_init(PLS_SO_EXTI_LINE, EXTI_INTERRUPT, EXTI_TRIG_RISING);

根因 B：g_plsState 在主循环中延迟读取，PLS 电平已变化

// 错误：MeasureTask 内才读引脚，电平可能已跳变
void MeasureTask(void) {
    g_plsState = gpio_input_bit_get(...);  // ← 读的是"现在"不是"中断时"
}

根因 C：delay_1ms(20) 扩大了跨轮 flag 积累窗口

① 上升沿EXTI → 填B → DMA完B → g_isB_Done=true ↓ delay 20ms 期间 ↓ 下降沿EXTI → g_plsState=LOW, g_isStartSampling=true ② 循环顶: MeasureTask 填A → DMA完A → g_isA_Done=true ↓ delay 20ms 期间 ↓ 上升沿EXTI → g_plsState=HIGH, g_isStartSampling=true ③ 循环顶: MeasureTask 填B（旧B正被覆盖！） → 检查: g_isA_Done=true && g_isB_Done=true ← 两flag均true！ A = ②本轮新数据 B = ①上轮旧数据，且DMA正在覆盖中！ → AnalyzeTask读B的同时DMA写B → 数据竞争 + 非配对
🔴 结论
delay 越长 → 跨轮 flag 积累窗口越大 → 整组错乱越频繁
P2

每组数据 [0] 位被前一轮 RDATA 残留污染

现象：无论整组是否正确，A[0] 或 B[0] 总是出现异常值。

📊 调试截图数据验证
A[0]: 7 ← B路基线值（B的RDATA残留）    A[1..9]: 547~534 ← A真实信号
B[0]: 533 ← A路信号值（A的RDATA残留）   B[1..9]: 7~8 ← B真实信号

根因：GD32 ADC 完成最后一次转换后，RDATA 寄存器保留旧值且 EOC 标志仍然置位（ADC DMA 请求由 EOC 驱动）。原 ADC_SwitchBuffer 末尾调用了 dma_channel_enable，此时 TIMER2 尚未启动，DMA Enable 后立即响应这个挂起的 EOC，将旧 RDATA 写入新缓冲区 [0]。

// 旧错误时序
ADC_SwitchBuffer(...):
    dma_memory_address_config(...)
    dma_channel_enable()   // ← DMA立即响应挂起EOC → 写[0] ← 污染！

ADC_StartSampling():
    dma_channel_disable()
    dma_channel_enable()   // ← 实际有效的Enable
    timer_enable(TIMER2)   // Timer触发真实采样 → 写[1..9]

3修复措施

Fix 1EXTI 改为双边沿触发
📄 BSP/src/bsp_exit.c
// 修改前
exti_init(..., EXTI_TRIG_RISING);

// 修改后
exti_init(..., EXTI_TRIG_BOTH);
Fix 2ISR 立即锁存 PLS 电平
📄 App/src/gd32c2x1_it.c
// 修改后：中断触发时刻立即锁存
void EXTI5_9_IRQHandler(void) {
    g_plsState = (gpio_input_bit_get(...))
                 ? kHigh : kLow; // ← 最准确的时刻
    g_isStartSampling = true;
}
Fix 3g_isSampleDone=false 提前到函数入口
📄 App/src/task_meas.c
void MeasureTask(void) {
    g_isSampleDone = false; // ← 最先执行，关闭重入
    ADC_SwitchBuffer(...);
    ADC_StartSampling();
}
Fix 4启动新路采样前清该路旧 Done 标志
📄 App/src/task_meas.c
if (g_plsState == kHigh) {
    g_isB_Done = false;  // ← 旧flag作废，防跨轮
    ADC_SwitchBuffer(g_B_Buffer);
} else {
    g_isA_Done = false;
    ADC_SwitchBuffer(g_A_Buffer);
}
Fix 5ADC_SwitchBuffer 不再 Enable DMA
📄 BSP/src/bsp_adc.c
void ADC_SwitchBuffer(int16_t *buf) {
    s_isFillingA = (buf == (int16_t *)g_A_Buffer);
    dma_channel_disable(ADC_DMA_CHANNEL);
    dma_memory_address_config(...);
    dma_transfer_number_config(...);
    // Enable 由 ADC_StartSampling 统一负责
}
Fix 6Enable DMA 前清 ADC EOC 标志
📄 BSP/src/bsp_adc.c
void ADC_StartSampling(void) {
    dma_channel_disable(...);
    dma_transfer_number_config(...);
    adc_flag_clear(ADC_FLAG_EOC); // ← 撤销挂起请求
    dma_channel_enable(...);       // 此时无挂起EOC，安全
    timer_enable(TIMER2);
}

5最终正确时序（修复后）

PLS上升沿 → EXTI ISR: g_plsState=HIGH, g_isStartSampling=true → 主循环: MeasureTask() g_isSampleDone = false // 关闭 EXTI 重入窗口（最先执行） g_isB_Done = false // 清除该路旧 flag，防跨轮配对 ADC_SwitchBuffer(g_B_Buffer) // 配置 DMA 地址（不 Enable） adc_flag_clear(ADC_FLAG_EOC) // 清除挂起的旧 EOC dma_channel_enable() // Enable DMA（此时无挂起请求） timer_enable(TIMER2) // Timer 触发真实采样 → DMA 完成中断: g_isB_Done=true, g_isSampleDone=true PLS下降沿 → EXTI ISR: g_plsState=LOW, g_isStartSampling=true → 主循环: MeasureTask() g_isSampleDone = false g_isA_Done = false // 清除该路旧 flag ADC_SwitchBuffer(g_A_Buffer) adc_flag_clear(ADC_FLAG_EOC) dma_channel_enable() timer_enable(TIMER2) → DMA 完成中断: g_isA_Done=true, g_isSampleDone=true 主循环检查: g_isA_Done && g_isB_Done → true（同一轮 A+B 均完成）✅ → g_isA_Done=false, g_isB_Done=false → AnalyzeTask() // 读缓冲区时 DMA 已停止，无竞争 → OutputTask() → LedTask()

6延伸：ADC 采样知识点总结

6.1 ADC EOC 标志与 DMA 挂起请求

⚠️ 关键规则
ADC 每完成一次转换，EOC 标志置位，同时向 DMA 发出请求。若此时 DMA 未 Enable， 这个请求会被硬件保持（pending），直到 DMA 被 Enable 后立即响应。

结论：Enable DMA 前必须用 adc_flag_clear(ADC_FLAG_EOC) 清除 EOC， 否则必然触发一次无效 DMA 传输写入缓冲区 [0]。 
// 正确时序
dma_channel_disable()
adc_flag_clear(ADC_FLAG_EOC)   // ← 必须在 Enable 前清除
dma_channel_enable()            // 无挂起请求，等待 Timer 真实触发
timer_enable(TIMER2)            // Timer → TRGO → ADC 转换 → DMA 写 [0]

6.2 Timer 触发 ADC 时序要求

正确： 配置DMA地址 → 清EOC → Enable DMA → Enable Timer ↑ Timer TRGO触发 → ADC转换 → 写[0] 错误： 配置DMA地址 → Enable DMA → Enable Timer ↑ 残留EOC立即驱动DMA写[0]（旧值）→ Timer触发写[1..9]

6.3 多缓冲区切换的并发安全三原则

  • 🔒
    "忙"标志置位时机要最早
    g_isSampleDone = false 应在任何共享资源操作（SwitchBuffer）之前，才能关闭中断重入窗口。
  • 🗑️
    启动采样前先清该路旧 Done 标志
    防止上一轮残留的 g_isX_Done=true 与本轮新采样的 g_isY_Done=true 发生跨轮误配对。
  • 🚫
    禁止在 DMA 传输期间重配目标地址
    必须先 dma_channel_disable 再修改 memory_addr，否则 DMA 内部指针与配置寄存器不一致。

6.4 EXTI 触发沿选择

场景建议触发沿
只关心信号到来（单向变化）EXTI_TRIG_RISING 或 FALLING
A/B 双相位切换（如 PLS_SO）必须 EXTI_TRIG_BOTH，否则一路相位永远不触发
防抖动要求高配合软件防抖或硬件滤波器

6.5 ISR 中应立即锁存引脚电平

💡 原则
所有与"触发边沿相关的状态"必须在 ISR 中立即记录。ISR 执行期间引脚电平最接近触发瞬间； 主循环中读取时，PLS 可能已跳变（尤其主循环周期远大于信号半周期时）。 
// 错误：主循环读时电平已变
void EXTI_Handler(void) { g_isStartSampling = true; }
void MeasureTask(void) { state = gpio_input_bit_get(...); } // ← 已变！

// 正确：ISR 内立即锁存
void EXTI_Handler(void) {
    g_plsState = gpio_input_bit_get(...) ? kHigh : kLow; // ← 最准确
    g_isStartSampling = true;
}

6.6 裸机主循环中 delay 的危害

  • ⏱️
    扩大中断积累窗口：delay 期间多个 EXTI 连续触发，中间请求丢失，g_isStartSampling 只保留最后一次。
  • 🔀
    允许 Done 标志跨轮积累：g_isA_Done 来自本轮，g_isB_Done 来自上轮，两者同时为 true 触发错误配对分析。
  • 💥
    造成数据竞争：分析任务读缓冲区时，新 DMA 已经在写同一块内存。
✅ 正确做法
裸机主循环中不使用阻塞 delay，改用非阻塞时间片（SysTick 计数比较） 或状态机节拍，保持主循环以最快速度轮询事件标志。

6.7 ADC 采样值的常用滤波方法

方法适用场景复杂度
算术平均噪声均匀，无明显毛刺低
截尾均值（Trimmed Mean）有偶发脉冲毛刺 （本项目采用，一级）低
中位数毛刺较多，样本量小低
最密集簇均值样本分散但有稳定核心 （本项目二级）中
IIR 低通实时流式采样，延迟可接受低
Kalman 滤波高精度，噪声模型已知高

本项目两级滤波链：

ADC 原始 10 点 → 截尾均值（去最大最小各1个，求平均）→ V_A / V_B → V_OUT = V_A - V_B，累积 20 组 → 聚类窗口8（最密集8个求均值）→ 最终 V_OUT 输出

6.8 双缓冲设计最佳实践（裸机）

推荐结构： 缓冲 A ──────► [分析任务读] 缓冲 B ──DMA──► [ADC写入中] A 满 → 通知分析任务 → 分析读 A → 同时 DMA 可安全写 B B 满 → 通知分析任务 → 分析读 B → 同时 DMA 可安全写 A 关键约束： 1. 分析任务读某缓冲时，DMA 绝对不能写同一缓冲 2. 切换缓冲前必须 dma_channel_disable，再改 memory_addr 3. 每次切换前清除目标缓冲的"完成"标志（防跨轮误配） 4. Enable DMA 前清 ADC EOC 标志（防 [0] 位污染）
